How Django works

June 14, 2020

 

My website has been created using a web framework called Django. Django is an open source high-level python script that takes care of the base website, allowing the main programmers focus to lie on writing the app.

 

 

In a Django driven website, the app waits for a HTTP request from the client before showing the content. The URL function (urls.py) processes the requests sent from the client and maps them to the appropriate ‘View’, webpage, by searching the string patterns created by the user. A URL mapper (urlpatterns) is responsible for creating paths to other areas in the website. In my website, I have created other HTML documents to expand on the website created in hand.

 

urlpatterns = [
   path( ' ', views.home, name="home"),
   path( 'blog.html', views.blog, name="blog"),
   path( 'blog1.html', views.blog1, name="blog1"),
]
  Using the data given by the URL function, views.py is the main body of the website, linking the HTTP responses to templates and models. All of these functions use render() which creates a HTTP request that is relayed back to the client. It links the HTML file back to the URL back to the client  
def home(request):
  return render(request, 'home.html',{})
def blog(request):
  return render(request, 'blog.html',{})
def blog1(request):
  return render(request, 'blog1.html',{})
  Models provides the function to read and write data, formatting them to a data base. Personally for my portfolio website, I didn’t add anything into this python file; as I had nothing to put into the database.
After being requested from views, the template file contains HTML (hypertext mark-up language); this is what the user controls in the website.

HTML file

 
My home page is contained in the file home.html, which is a HTML5 file. Before creating this website I felt quite confident in HTML programming so I had high ambitions. Following off templates, I knew how I wanted the website to look.
I used a modular based approach for the website, where each scroll down had its own Div class, this made it easier to edit the code; and add pre-sets in.
For my blog page I wanted to add modals, a graphical interface that goes over the applications main window; I found this the most challenging part of creating this website. First off I created a simple modal, using the built in HTML modal class; this worked well for just one modal however, on my home screen I needed multiple amounts of modals to be shown. Instead of copying the code again and changing the code for each of the ‘learn more’ buttons, I went back to the drawing board. I used a method called .getElementsByClassName, in Javascript, which searches the whole document for the phrase indented in the brackets; and gives each of the functions a numerical value.
var slidesJS = document.getElementsByClassName("blog-preview");
The selection of modals is processed through the slideIndex number, which looks at which modal expansion is pressed. This is an example of the code used with the ‘onclick’ button function. The modal gets opened with the 2nd modal content.
openModal();currentSlide(2)
Another difficulty faced was the ‘overflow’ function, which disables the scroll feature when the modal is open. Typically modals are small overlays that don’t cover the whole screen however, my modals contained a whole blog that required scrolling. Therefore, with scrolling disabled I could neither scroll on the main page or the overlay. After many hours spent on stack overflow, I found that I needed to use a different display style with the modal but also had to change the body’s documentation to allow this change in users actions.
My website also contains a downloadable CV and a contact me page, that sends an email to my personal email address.

Using CSS files

 
CSS stands for Cascading Style Sheets, and controls the layout of the web page. They are used for nearly every design element of making a website. For my website I showed all three ways to use CSS, Inline, Internal and External. Inline, uses ‘style’ on the same line as the words
style="color:black ;"
Internal CSS was used in my HTML for button styling and modal styling, this could’ve been added to an external CSS file however, I decided to put it in my main file for ease of use when creating the modals.
"style"
Finally, External Cascading sheets are used for the majority of the styling.
body {
    font-family:'Lato', sans-serif;
    overflow:hidden;
    overflow-y:scroll;
    background-color: #000;
}

For my website, I used some templates to give the effect I was after; such as PixelCog’s Parallax and lots of bootstraps javascript. This was relatively easy to implement and I had to adjust some of the values to toggle them into position but overall worked well.

Reflecting on First year

June 10, 2020

 

My first year at the University of Birmingham has come to an end, and although it was short lived; thank you corona, I can’t emphasise how brilliant it was! It only seems like yesterday I was arriving at the university with a car full of new and clean IKEA kitchen ware. Trust me, I went a bit overboard!
It’s equally as scary thinking I’ll be a second year student next year, where the work ramps up exponentially. Freshers was amazing. I met loads of new people and spent the whole time socialising with everyone. This was the perfect opportunity to join new societies as well, so I joined the go-karting team, the ski-team and also one of the university’s campus football teams. A lot of sport! This was also the first time where I had lived away from home, so the challenge of adulthood really stepped in quickly.

 

First term

Social side aside, I’ve loved the topics I had this year in Computer science. We started the term learning Java script, and although this was a challenge for me at the start, as I had not done computing A-level, after a few weeks I got to grip of it and excelled through the tasks. I learnt a lot during this module, such as the fundamental lesson that planning before starting a project is always the way to go. Collaboration really helped me as well, as in some tasks we got given partially competed code, where we had to continue the coding method to expand this pre-made program.

 

Alongside the Java aspect of the course, we also had Mathematics tuition. This module focused on the mathematics behind Computer Science, covering a range of topics from regular languages to the complexity of NP and P problems. I had to think quite hard in this module, reading around the course to get a full grasp about the power of these mathematic equations.

 

As a Bachelor of Science degree apprenticeship student with Vodafone, I was required to study Finance and Business Management as part of my widening horizons modules. Their goal was to give me insight into how organisations are structured; from how they work their balance sheet to understanding EBITDA formulas . It really helped having past experience in Economics for this module, as I could implement everything I learnt for a level into my end of year dissertation.

Second term

The second term consisted of Robotics, Artificial intelligence and Logic & Computation. I loved Robotics because we could see our code running on a machine, instead of just a computer. Hardware vs Software was a problem for the robots as we would code a piece of software that would complete every task when tested in a software sandbox but fail in a real-life. This involved a lot of trial and error but I thoroughly enjoyed this process.

 

I found Artificial Intelligence the hardest topic in Computer Science, as I was introduced to many alien terms. It took time to fully understand the ideas around machine learning and all of the equations behind it.

 

Logic and Computation took a lot of time to fully understand but as the name suggests, when fully understood, was quite logical! The complexity of the topics increased every week and it was essential that I understood the basic ideas in order to extend my thinking.
